Sheet mechanics per platform

  • Android (Compose/M3): ModalBottomSheet with drag handle; system/predictive back dismisses; IME pushes the sheet up.
  • iOS (SwiftUI): .sheet with .presentationDetents([.medium,.large]) + grabber; swipe-down dismisses.
  • Dismissal (either platform) preserves entered data — the §3.4 wizard model persists; Back/Esc never loses input.
  • HarmonyOS (ArkTS): bindSheet panel · Aurora (Qt Quick): Silica Dialog page pushed from the pulley menu — SCAFFOLD [GAP: 8.5].

Forms mandate (wireframes §1.3)

  • Hint is wired via aria-describedby before the error; the error line is instructive: it names the failing value (“example.com/mlpapers”), the reason (not a t.me/max.ru link → 422) and the fix (an example valid link).
  • Errors use --danger, never --accent; aria-invalid="true" on the field; Add channel stays disabled until Resolve succeeds (per-step gating, §3.4 table).
  • Sign-in sub-flow (step 2 “+ sign in another”) is the ux-flows §2.1 state machine — interactive phone + code + 2FA, or non-interactive TG_*/MAX_* env vars.
